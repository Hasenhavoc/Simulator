📖 How Each Model Works (click to expand)
━━ CONSTANT MODELS (no trend, no seasonality)
Simple Average: Mean of all historical months. Forecast = Σhistory/N. Ignores recent trends. Use only for very stable demand. Formula: F = (150+144+156+...)/24 = 182u.
SMA (Simple Moving Average): Average of last 3 months only. More responsive than full average. F = (month₋₁ + month₋₂ + month₋₃)/3.
SES (Simple Exponential Smoothing): Weighted blend — recent data weighted more. α=0.3 means 30% weight on newest observation. F[t] = α×actual[t] + (1-α)×F[t-1]. Your α slider controls this.
Adaptive SES: α adjusts automatically based on tracking signal. If forecast consistently overshoots, α increases (more reactive). α = |smoothed_error / smoothed_abs_error|, clamped [0.05, 0.95].
━━ TREND MODELS (trend, no seasonality)
DES (Double Exponential / Holt's): Tracks level AND trend separately. Good when demand is consistently growing or declining. Level: L[t] = α×actual + (1-α)×(L+T). Trend: T[t] = β×(L[t]-L[t-1]) + (1-β)×T[t-1]. Forecast: L + h×T.
Linear Regression: Fits a straight line through all data. Slope = trend rate. F[t] = intercept + slope×t. Simple but misses seasonality and recent changes.
Croston's Method: For sporadic/intermittent demand (many zeros). Separately forecasts probability of demand occurring AND size when it occurs. Used for spare parts, slow-moving SKUs. Not relevant for our cake bakery.
━━ SEASONAL MODELS (the ones that usually win)
HW-Multiplicative (Holt-Winters): Tracks Level + Trend + Seasonal index. Seasonal effect is a MULTIPLIER (Dec=1.65×, Feb=0.82×). L[t] = α×(actual/S[t-12]) + (1-α)×(L+T). S[t] = γ×(actual/L) + (1-γ)×S[t-12]. F[t+h] = (L+h×T) × S[t+h-12]. This is what P&G, Unilever, Nestlé use for FMCG.
HW-Additive: Same as HW-Mult but seasonal effect is ADDED not multiplied. Better when seasonal swings are constant in absolute terms (always ±50u, not ±30%). F = (L+h×T) + S instead of × S.
Seasonal Linear Regression: Fits trend line + 12 monthly dummy variables. Each month gets its own intercept adjustment. F = a + b×t + c₁×Jan + c₂×Feb + ... + c₁₂×Dec.
Auto-SARIMA: ARIMA with seasonal differencing. Auto-selects (p,d,q)(P,D,Q,m) by testing combinations. The academic gold standard. Amazon and Walmart use this in automated pipelines. Our implementation: d=1, seasonal period=12, AR from PACF.
━━ ML MODELS
Gradient Boosting (GBM): 50 decision trees, each correcting the previous tree's errors. Features: month index (captures trend) + month-of-year (captures seasonality). Learning rate=0.1, max depth=3. Powerful but needs 24+ months to avoid overfitting. Used by Amazon, Alibaba.
━━ MODEL SELECTION (how the winner is chosen)
Walk-forward validation: Train on first N-6 months, forecast last 6 months. Compare forecast vs actual. Score = 0.4×MAPE + 0.6×WMAPE. Pattern penalties: constant model on seasonal data → ×1.25 penalty. Bonuses: seasonal model on seasonal data → ×0.95. The lowest score wins. This prevents SES from "winning" just because it happens to fit the holdout despite ignoring seasonality.

🧮 MILP FORMULATION — Full Mathematical Model (click to expand)
OBJECTIVE FUNCTION — Minimize Total Cost
min Z = Σt=1..T [ S·sw[t] + hfg·I[t] + c·p[t] + w·e[t] + b·s[t] + λ·ssv[t] + f ] + ΣiΣt hrm,i·RI[i,t]
S·sw[t] = setup cost $50 × batch SWITCH flag. Fires ONLY when production starts after idle — not every day. Factories run daily; sw[t]=1 means y[t]=1 but y[t-1]=0.
hfg·I[t] = FG holding $0.0216/unit/day. Landed cost of all BOM parts × carry rate ÷ 365 + labor holding.
c·p[t] = unit production cost $33 = (material + labor) ÷ yield. You PAY for inputs but GET only yield% output.
w·e[t] = waste cost $6 per expired unit = material value × (1 - salvage%).
b·s[t] = shortage cost $100/unit. Backorder OFF → lost sale at sell price. ON → penalty per unit delayed.
λ·ssv[t] = safety stock violation penalty $0.065/unit/day below SS. SS is a TARGET, not a wall — can dip below during spikes.
f = fixed daily overhead $100 — paid regardless of production.
hrm,i·RI[i,t] = RM holding per subpart. Each subpart charged at its own cost × carry%/365.
T = 261 working days. Not 365 — calendar-correct per month.
DECISION VARIABLES
p[t] = FG units to produce on day t (integer, ≥0) — main solver output
y[t] = 1 if production line is active on day t (binary) — links to capacity
sw[t] = 1 if batch STARTS on day t (binary) — sw[t] ≥ y[t] - y[t-1]. Triggers setup cost only on transition from idle→active.
I[t] = FG inventory at end of day t (continuous, ≥0). I[0] = 0 (your starting on-hand).
e[t] = units expired on day t (forced by shelf life constraint)
s[t] = shortage on day t. Backorder OFF: forced to 0. ON: carries to t+1 as extra demand.
ssv[t] = how far below safety stock on day t (continuous, ≥0). Penalized but allowed.
r[i,t] = RM units of subpart i ordered on day t (integer) — arrives at t+LT[i]
RI[i,t] = RM inventory of subpart i at end of day t
zo[i,t] = 1 if PO placed for subpart i on day t (binary) — links to MOQ
CONSTRAINTS (12 total)
C1. FG Flow Balance: I[t] = I[t-1] + p[t]×yield - d[t] - e[t] + s[t]
Backorder ON: I[t] = I[t-1] + p[t]×yield - d[t] - s[t-1] - e[t] + s[t] (yesterday's backorder adds to today's demand)
C2. Capacity: p[t] ≤ C[t] · y[t]
If p[t]>0, forces y[t]=1. C[t] = 21 u/day (cycle×shift×util already applied).
C3. Batch Switch: sw[t] ≥ y[t] - y[t-1]
sw=1 ONLY when starting after idle. Continuous production = 1 switch total. sw[0] ≥ y[0] (first day always a switch if producing).
C4. Warehouse: I[t] ≤ WHmax (500u)
C5. Safety Stock (soft): ssv[t] ≥ SS - I[t]
SS=6u. If inventory drops below, ssv[t] captures the gap. Penalized at $λ/unit/day — NOT a hard wall.
C6. No Backorder: s[t] = 0 (when backorder OFF)
Forces all demand to be met or lost. When ON: s[t] ≥ 0 freely, penalized at $b/unit.
C7. Shelf Life Expiry: Σe[τ] ≥ Σ(p[τ]×yield) - Σd[τ] for τ in aged window
Inventory produced >shelf days ago and not yet sold MUST expire. Shelf=3d.
C8. BOM Explosion: r[i,t-LT[i]] × partYield[i] ≥ p[t] × qty[i]
RM ordered LT days ago × subpart yield must cover production × BOM qty. This IS the MRP explosion — built into the solver.
C9. RM Flow Balance: RI[i,t] = RI[i,t-1] + r[i,t] - p[t]×qty[i]
RM stock = previous + received - consumed. Consumed instantly when production happens.
C10. RM Warehouse: RI[i,t] ≤ RMmax[i]
Per-subpart physical storage limits.
C11. MOQ: r[i,t] ≥ MOQ[i] × zo[i,t]
If ordering (zo=1), must order at least MOQ. If not (zo=0), r=0.
C12. Supplier Max: r[i,t] ≤ MaxOrder[i] × zo[i,t]
Can't order more than supplier can deliver per shipment.
WHEN TO DROP / SIMPLIFY
No shelf life? → Drop C7, e[t]. Classical lot-sizing.
No MOQ? → Drop C11-C12, zo[i,t]. Continuous ordering.
No backorder? → C6 forces s[t]=0. Remove s[t] from C1.
Infinite capacity? → Drop C2, y[t], sw[t]. Becomes LP.
Parallel lines? → Drop shared capacity. Each SKU gets own C2: p[k,t] ≤ C_k[t].
SIMPLEX vs BRANCH & BOUND
LP (Simplex): No binary variables (y,sw,zo removed). All continuous. Polynomial time. Use when S=0, no MOQ.
MILP (B&B): y[t], sw[t], zo[i,t] are binary. NP-hard. CBC: ~2min for 261d/2 parts. Gurobi: ~10s.
